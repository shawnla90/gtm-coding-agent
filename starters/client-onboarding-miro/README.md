# Client Onboarding Miro Starter

This starter captures the Chapter 17 onboarding-board pattern.

Use it when you want to map a client's GTM engine before kickoff: sources,
database, enrichment, CRM, outreach, website, content, funnel, and measurement.
The files here are intentionally small so you can copy them into a client
workspace and adapt them with a coding agent.

## Files

- `board-template.json` - offline board template with column coordinates,
  colors, and connector intent styles.
- `pipeline-6-column.yaml` - six-column website/content pipeline layout.

## Operating rule

If a value is unknown, write `NEEDS INPUT`. Do not invent the client's tools,
costs, owners, credentials, or API scopes to make the board look complete.

## Visual QA

The Miro API returning `200 OK` only proves that the request succeeded. It does
not prove that the board is readable. Always screenshot the board and inspect
for overlaps, clipped text, stacked shapes, and connectors routed through boxes.
